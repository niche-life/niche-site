# 画像処理ガイドライン

## 表紙画像仕様

### 基本要件
- **形式**: JPEG (.jpg)
- **サイズ**: 180×254px（推奨）
- **縦横比**: 1:1.41（A4比率に近似）
- **ファイルサイズ**: 100KB以下
- **品質**: 80-90%

### 命名規則
```
volume-{番号}.jpg
```
例: `volume-1.jpg`, `volume-13.jpg`

## 画像最適化手順

### 1. 元画像の準備
- 高解像度の原稿から切り出し
- 縦横比を1:1.41に調整
- 不要な背景やロゴを除去

### 2. リサイズと最適化
```bash
# ImageMagickを使用した最適化例
magick input.jpg -resize 180x254! -quality 85 volume-13.jpg

# 代替手段（オンラインツール）
# TinyPNG, Squoosh.app等を使用
```

### 3. 品質チェック
- [ ] ファイルサイズが100KB以下
- [ ] テキストが読みやすい解像度
- [ ] 色彩が適切に保持されている

## アイコン・ロゴ画像

### サイトアイコン
- **場所**: `src/assets/icon.png`
- **サイズ**: 512×512px
- **形式**: PNG（透明背景推奨）

### 編集委員画像
- **場所**: `src/assets/editors/`
- **サイズ**: 150×150px（正方形）
- **形式**: JPEG
- **命名**: `lastname-firstname.jpg`

## パフォーマンス考慮事項

### WebP対応
Astroが自動的にWebP変換を行うため、ソース画像はJPEGで問題なし。

### 遅延読み込み
`<Image>`コンポーネントが自動的に遅延読み込みを適用。

### レスポンシブ対応
```astro
<Image
  src={coverImage}
  alt={`${volume.title}の表紙`}
  width={180}
  height={254}
  class="w-full max-w-[180px] h-auto"
/>
```

## バックアップと管理

### ファイル管理
- 元画像は別途保管
- 最適化済み画像のみをリポジトリに含める
- 定期的な画像監査（年2回推奨）

### 品質管理指標
- 読み込み時間: 1秒以下
- Lighthouse画像スコア: 90点以上
- Core Web Vitals: LCP 2.5秒以下
