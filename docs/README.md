# ニッチェ・ライフ ドキュメント構成

このプロジェクトの設定ファイルとドキュメントの概要です。

## 主要設定ファイル

### `CLAUDE.md`
Claude Codeとの連携用の基本設定ファイル。プロジェクト概要、コーディング規約、コマンドリファレンス、重要な注意事項を記載。

### `.claude/rules/`
特定のファイルパターンに適用される詳細ルール：
- `astro-components.md`: Astroコンポーネント開発規約
- `data-management.md`: データ管理・整合性ルール

## ドキュメント群

### `docs/project-overview.md`
プロジェクト全体のビジョン、ターゲットユーザー、サイト構造の理想形を記載。新しいメンバーや協力者向けの導入資料。

### `docs/submission-workflow.md`
新巻号追加の具体的手順書。編集委員会メンバーが新しい号を公開する際のステップバイステップガイド。

### `docs/image-guidelines.md`
表紙画像や各種画像の処理・最適化ガイドライン。品質基準とパフォーマンス要件を明記。

### `docs/seo-checklist.md`
SEO最適化の具体的チェック項目。学術雑誌としての構造化データ設定も含む。

## 使い方

### 開発者向け
1. `CLAUDE.md`を確認してプロジェクトの基本を理解
2. `.claude/rules/`で該当する作業領域のルールを確認
3. 必要に応じて`docs/`の詳細ドキュメントを参照

### 編集委員向け
1. `docs/project-overview.md`でサイトの目的を確認
2. `docs/submission-workflow.md`に従って新巻号を追加
3. `docs/image-guidelines.md`で画像処理方法を確認

### 運用・管理者向け
1. 定期的に`docs/seo-checklist.md`の項目を実施
2. パフォーマンス監視とユーザビリティ改善
3. 新しい要件に応じてドキュメントを更新

## 更新方針

これらのドキュメントは実際の運用から得た知見を反映して継続的に更新します：
- 問題が発生した際の解決策をドキュメント化
- 新機能追加時の手順書更新
- ユーザビリティテストの結果を反映
